## Endpoints
**GET** `forecast/{date}`<br/>
Gets the weather forecast for a specific date

## Parameters
### Path Parameters
<ParamField path="{date}" type="string" required="true">
  The date for the weather forecast. Must be formatted as YYYY-MM-DD.
</ParamField>

## Sample Request
<CodeGroup>

```curl cURL
curl -I -X GET "https://api.openweathermap.org/com/forecast/2024-12-25?appid=APIKEY"
```

```javascript JavaScript - jQuery
var settings = {
  "url": "https://api.openweathermap.org/com/forecast/2024-12-25?appid=APIKEY",
  "method": "GET",
  "timeout": 0,
};

$.ajax(settings).done(function (response) {
  console.log(response);
});
```

```http HTTP
GET /com/forecast/2024-12-25?appid=APIKEY HTTP/1.1
Host: api.openweathermap.org
```

```nodejs NodeJs
var request = require('request');
var options = {
  'method': 'GET',
  'url': 'https://api.openweathermap.org/com/forecast/2024-12-25?appid=APIKEY',
  'headers': {
  }
};
request(options, function (error, response) {
  if (error) throw new Error(error);
  console.log(response.body);
});
```

```ruby Ruby
require "uri"
require "net/http"

url = URI("https://api.openweathermap.org/com/forecast/2024-12-25?appid=APIKEY")

https = Net::HTTP.new(url.host, url.port)
https.use_ssl = true

request = Net::HTTP::Get.new(url)

response = https.request(request)
puts response.read_body
```
</CodeGroup>
In the above requests, replace `APIKEY` with your actual API Key value.

## Response
`application/json`

<Tabs>
  <Tab title="Sample Response">
    ```json
{
"date": "2015-09-01",
"description": "sunny",
"maxTemp": 22,
"minTemp": 20,
"windSpeed": 12,
"danger": false
}
```

  </Tab>
  <Tab title="Response Definitions">
    <ResponseField name="date" type="string">
  The date for the weather forecast.
</ResponseField>
<ResponseField name="description" type="string">
  Text description of the weather. 5 values are possible:
- `sunny`
- `overcast`
- `partly cloudy`
- `raining`
- `snowing`
</ResponseField>
<ResponseField name="maxTemp" type="number">
  The highest temperature (in Celsius) for the date.
</ResponseField>
<ResponseField name="minTemp" type="number">
The lowest temperature (in Celsius) for the date.
</ResponseField>
<ResponseField name="windSpeed" type="number">
The wind speed for the date.
</ResponseField>
<ResponseField name="danger" type="boolean">
Indicates whether the weather conditions are dangerous for the date.
</ResponseField>
  </Tab>
</Tabs>
### Response Codes
200 - Request successful. <br />
400 - Bad Request. A provided error message will explain further.
